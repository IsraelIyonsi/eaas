Quickstart (Node.js + cURL)

SendNex is a plain HTTP JSON API. You can hit it with any HTTP client. Below are the two fastest paths — a one-line cURL and a Node.js snippet.

Base URL: https://api.sendnex.xyz

cURL

# Send your first email
curl https://api.sendnex.xyz/api/v1/emails/send \
  -H "Authorization: Bearer sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "from": "noreply@yourapp.com",
    "to": ["alice@example.com"],
    "subject": "Welcome to YourApp",
    "html": "<h1>Hi Alice</h1>"
  }'

Node.js (fetch)

const res = await fetch("https://api.sendnex.xyz/api/v1/emails/send", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.SENDNEX_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    from: "noreply@yourapp.com",
    to: ["alice@example.com"],
    subject: "Your receipt",
    html: "<p>Thanks for your order</p>",
  }),
});
const data = await res.json();
console.log(data.id);

Authentication (API keys)

Every request must include an API key in the Authorization header using the Bearer scheme. You can create, rotate, and revoke keys from app.sendnex.xyz/api-keys.

  • Live keys — prefix sk_live_ — send real email, counted against your billing plan.
  • Test keys — prefix sk_test_ — accepted by the API but no email leaves our network. Great for CI.

Keep keys in a secrets manager or environment variable. Never commit them to source control.

Rotate a key by creating a new one, deploying it, then deleting the old one. Deleted keys are rejected immediately.

Send your first email

Before your first send, verify the domain you will send from. SendNex generates the DKIM, SPF, and DMARC records for you — paste them into your DNS and we check propagation automatically.

  1. Create an account at app.sendnex.xyz/signup.
  2. Add a sending domain and publish the DNS records we show you.
  3. Create an API key.
  4. Call POST /api/v1/emails/send with from, to, subject, and one of html or text.
  5. Track the result in the dashboard, or via a webhook (see below).

Minimum valid request body:

{
  "from": "noreply@yourdomain.com",
  "to": ["recipient@example.com"],
  "subject": "Hello from SendNex",
  "text": "Your first SendNex email."
}

Webhooks

Register a webhook URL from the dashboard and SendNex will POST a JSON event every time something interesting happens to one of your messages. Each request is signed with HMAC-SHA256 using your webhook secret — verify the signature before trusting the payload.

Supported event types:

  • email.sent — accepted by the outbound MTA
  • email.delivered — the recipient server accepted the message
  • email.bounced — hard or soft bounce (check reason)
  • email.complained — recipient reported spam
  • email.opened / email.clicked — engagement (if tracking is on)
  • email.received — inbound parse fired on your domain

Example handler (Node.js / Express):

import crypto from "node:crypto";

app.post("/sendnex-webhook", express.raw({ type: "application/json" }), (req, res) => {
  const sig = req.header("X-SendNex-Signature") ?? "";
  const expected = crypto
    .createHmac("sha256", process.env.SENDNEX_WEBHOOK_SECRET)
    .update(req.body)
    .digest("hex");
  if (!crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected))) {
    return res.status(401).end();
  }
  const event = JSON.parse(req.body.toString("utf8"));
  console.log(event.type, event.data.id);
  res.status(200).end();
});

Webhooks retry with exponential backoff for 24 hours on any non-2xx response. Return a 2xx as soon as you have persisted the event — do the slow work in a background job.

Code Meaning What to do

202AcceptedEmail queued; check webhooks for delivery.
