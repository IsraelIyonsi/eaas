# EaaS Coding Standards (Split)

Standards are split by topic. Read only the file relevant to your current task.

| File | When to read |
|---|---|
| `backend-structure.md` | Adding new features, creating entities, EF configs |
| `backend-cqrs.md` | Writing commands, queries, handlers, validators, endpoints |
| `backend-services.md` | Constants, service registration, API responses, error handling, messaging |
| `backend-testing.md` | Writing unit tests, validator tests |
| `backend-migrations.md` | Database schema changes |
| `frontend-architecture.md` | Repository pattern, React Query hooks, constants, types |
| `frontend-components.md` | Page patterns, shared components, styling, API response handling |
| `frontend-testing.md` | Playwright E2E tests, mock API |
| `infrastructure.md` | Docker, database config |
| `review-checklist.md` | PR review verification |
