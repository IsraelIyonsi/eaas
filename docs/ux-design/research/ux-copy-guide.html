Voice and Tone Principles

EaaS is a developer tool. The voice is technical, direct, and confident. No marketing fluff, no unnecessary words. Every message should help the user take action or understand status.

Direct

Say what happened and what to do next. No filler words.

Do: "Domain verified. You can now send from this domain."
Avoid: "Great news! Your domain has been successfully verified and is now ready for use!"

Technical

Use precise terminology. The user is a senior developer.

Do: "550 5.1.1 - Mailbox does not exist (hard bounce)"
Avoid: "This email could not be delivered to the recipient."

Actionable

Every error message must suggest what to do next.

Do: "DKIM record not found. Add the CNAME record below to your DNS provider."
Avoid: "Verification failed."

Honest

Show exact values, expected vs actual. No ambiguity.

Do: "Expected: v=spf1 include:amazonses.com ~all | Found: v=spf1 ~all"
Avoid: "Your SPF record appears to be incorrect."

Consistent

Same action = same verb everywhere. Same object = same noun.

Do: Always "Create" for new, "Save" for update, "Delete" for remove.
Avoid: Mixing "Add", "New", "Create", "Make" for the same action.

Concise

Fewer words is better. If it can be a status badge, it should be.

Do: "Delivered" badge, "2.1% bounce rate"
Avoid: "This email was successfully delivered to the recipient's mail server."

Empty States

Each empty state includes: an icon or illustration, a headline, a description, and a primary call to action. The tone is helpful and instructional, not apologetic.

Email Logs - No Emails
No emails sent yet

When you send your first email through the API, it will appear here with full delivery tracking.

Button: View API Documentation
Shown when the email logs table has zero entries. Primary CTA directs to Swagger docs.
Email Logs - No Results (Filtered)
No emails match your filters

Try adjusting the date range, status, or search terms.

Button: Clear Filters
Shown when filters return zero results but the table has data.
Templates - No Templates
No templates yet

Templates let you reuse email layouts with dynamic variables. Create your first template or use the API to send raw HTML.

Button: Create Template
Shown when the templates list is empty. Explains the value of templates.
Domains - No Domains
No sending domains configured

Add a domain to start sending emails. You will need to add SPF, DKIM, and DMARC records to your DNS provider.

Button: Add Domain
Shown when no domains are registered. Makes DNS requirement clear upfront.
Analytics - No Data
No analytics data available

Analytics will populate once emails start being sent. Data is aggregated daily.

Button: View Email Logs
Shown when analytics API returns empty data for the selected range.
Suppressions - Empty List
No suppressed addresses

Addresses that hard bounce or receive complaints are automatically added here. You can also manually suppress addresses.

Button: Add Suppression
Shown when suppression list is empty. Explains auto-suppression behavior.
API Keys - No Keys
No API keys created

Create an API key to authenticate your applications. Each key can be scoped to specific sending domains.

Button: Create API Key
Shown when no API keys exist. Unlikely scenario since a seed key bootstraps the system.
Template Version History - No Versions
No version history

Version history will appear after you save changes to this template.
Shown in the version history panel for newly created templates with only one version.

Verb Consistency Rules

New resource that did not exist: templates, API keys
Add
Register/include an existing thing: domains, suppressions

Persist changes to an existing resource: template edits

Verify
Check correctness: domain DNS records
Export
Download data: CSV files
Rotate
Replace with new while keeping old temporarily: API keys
Restore
Recover a soft-deleted resource: templates

Tooltip Text

Tooltips appear on hover over stat cards, status badges, and informational icons. They provide brief context or definitions. Max length: 120 characters.

Overview Stat Card Tooltips
Emails Sent
Total emails accepted by the API and enqueued for delivery in the selected period.
Delivery Rate
Percentage of sent emails confirmed delivered by the recipient's mail server.
Bounce Rate
Percentage of sent emails permanently rejected. Keep below 2% to protect sender reputation.
Complaint Rate
Percentage of delivered emails marked as spam by recipients. SES suspends at 0.1%.
Open Rate
Percentage of delivered emails where the tracking pixel was loaded. Undercounts due to image blocking.
Click Rate
Percentage of delivered emails where at least one tracked link was clicked.
System Health Tooltips
API Status
HTTP API server health. Green when responding within normal latency.
Worker Status
Background worker processing the email queue. Red means emails are queued but not being sent.
Queue Depth
Number of emails waiting to be processed. High values indicate a processing backlog.
PostgreSQL
Primary database for logs, templates, domains, and API keys.
Redis
Cache layer for rate limiting, suppression lookups, and template caching.
RabbitMQ
Message queue for async email processing with retry and dead-letter support.
Domain Page Tooltips
SPF Record
Sender Policy Framework. Authorizes mail servers to send on behalf of your domain.
DKIM Records
DomainKeys Identified Mail. Cryptographic signatures proving emails are unaltered.
DMARC Record
Domain-based Message Authentication. Tells mail servers what to do with failed SPF/DKIM checks.
Template Editor Tooltips
Variables Schema
JSON Schema defining expected template variables. Validated at send time to catch missing data.
Version History
Last 10 saved versions of this template. Compare changes between versions.
Send Count
Total number of emails sent using this template across all time.

Loading State Guidelines
Pattern
Use skeleton loaders (pulsing gray rectangles) for table rows and stat cards during initial page load.
Pattern
Use inline spinner + text for action confirmations (save, verify, create).
Pattern
Disable the triggering button during action loading to prevent double-submission.
Pattern
If loading takes more than 10 seconds, show: "This is taking longer than expected. Please wait..."
Pattern
If loading fails after timeout (30s), show the relevant error message from the Errors section.
