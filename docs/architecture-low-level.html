Processing Pipeline

1
Load Email
Fetch Email entity from PostgreSQL by ID
↓
2
Render Template
If template_id set, render Liquid template via Fluid engine with variables
↓
3
Inject Tracking
If track_opens: inject 1x1 pixel. If track_clicks: rewrite <a> hrefs via AngleSharp
↓
4

Simple format (no attachments) or MimeKit MIME message (with attachments/CC/BCC)
↓
5
SES Delivery
Call SendEmailAsync (simple) or SendRawEmailAsync (MIME). Update status to Sending
↓
6
Log Event
Create EmailEvent (Sent). Store SES Message-ID for bounce correlation

🔄Retry Policy1s -> 5s -> 30s intervals (MassTransit UseMessageRetry)
☠Dead Letter Queueeaas-emails-send_error after all retries exhausted

Update email status to Delivered, set DeliveredAt timestamp. Log EmailEvent.

POST /api/v1/emails/send - Complete Request Lifecycle

1
API Gateway
POST /api/v1/emails/send arrives at Nginx.
TLS terminated. Rate limit check at edge. Proxied to API :8080.
↓
2
ApiKeyAuthHandler
Extract Bearer token from Authorization header.
SHA-256 hash the token.
Lookup hash in Redis cache (fast path).
On miss: query PostgreSQL api_keys table, cache result.
Validate key is Active (or Rotating with valid expiry).
Set ClaimsPrincipal with TenantId + ApiKeyId.
↓
3
SendEmailEndpoint
Parse JSON request body into SendEmailCommand.
Extract TenantId and ApiKeyId from authenticated claims.
Dispatch via MediatR: mediator.Send(command)
↓
4
ValidationBehavior (MediatR Pipeline)
FluentValidation runs SendEmailValidator:
- Valid email format (from, to, cc, bcc)
- Total recipients (to + cc + bcc) ≤ 50
- Subject required, max 998 chars
- HTML or Text body required
- Attachments: max 10 files, max 7MB total
On failure: throws ValidationException -> 422 response.
↓
5
SendEmailHandler (Business Logic)
5a. Check rate limit - Redis sorted set + Lua script. 100 emails/min per key.
5b. Check idempotency key - Redis SET NX with TTL. Reject duplicate.
5c. Verify sending domain - PostgreSQL lookup: domain must be Verified status.
5d. Check suppression list - Redis cache first, PostgreSQL fallback. Check ALL recipients.
5e. Create Email entity - PostgreSQL INSERT with status = Queued.
5f. Publish SendEmailMessage - MassTransit -> RabbitMQ queue: eaas-emails-send.
5g. Return 202 Accepted with message_id.
↓ Queue: eaas-emails-send
6
SendEmailConsumer (Worker Service)
6a. Load Email entity from PostgreSQL by EmailId.
6b. If TemplateId set: render Liquid template via Fluid engine.
6c. If TrackOpens: inject tracking pixel before </body>.
6d. If TrackClicks: rewrite <a href> tags via AngleSharp.
6e. If attachments: build MIME message via MimeKit.
6f. Call AWS SES: SendEmailAsync (simple) or SendRawEmailAsync (MIME).
6g. Update Email: status = Sending, store SES Message-ID.
6h. Insert EmailEvent (EventType.Sent).
6i. On failure: MassTransit retries 1s -> 5s -> 30s -> DLQ.
↓ AWS SES Delivery
7
Post-Delivery Events (Async)
SES delivers to recipient mail server.
SES publishes event to SNS topic.
SNS POSTs to WebhookProcessor /webhooks/sns.

Delivery: Update status -> Delivered, set DeliveredAt.
Bounce (hard): Update status -> Bounced, auto-suppress recipient in DB + Redis.
Complaint: Update status -> Complained, auto-suppress in DB + Redis.
Open: Pixel loaded -> log EmailEvent (Opened), set OpenedAt.
Click: Link clicked -> log EmailEvent (Clicked), set ClickedAt, 302 redirect.
