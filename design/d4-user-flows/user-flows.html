Flow 4

Developer Webhook Integration

Implement webhook handler for inbound emails in their application

Persona: Alex (Developer)
Complexity: High
Screens: API Docs, Webhook Guide, Dashboard (debugging)
Sprint: 5
START: Alex reads API Documentation
Navigate to Webhook Integration Guide
Read Payload Schema
  • Understand JSON structure
  • Note HMAC-SHA256 signature in X-EaaS-Signature header
  • Note timestamp for replay protection
Choose language: Node.js / Python / C# / Go
Copy webhook handler code snippet
Implement in Application
7 Implementation Steps
  • Step 1: Create endpoint → POST /webhooks/email
  • Step 2: Verify HMAC signature (extract header, compute HMAC-SHA256, timing-safe compare)
  • Step 3: Check timestamp → Reject if > 5 minutes old
  • Step 4: Check idempotency → Store message ID, skip duplicates
  • Step 5: Acknowledge fast → Return 200 within 500ms
  • Step 6: Enqueue for processing (Redis, RabbitMQ, SQS)
  • Step 7: Process async (parse email, thread matching, download attachments, business logic)
Test with ngrok
  • Run ngrok http 3000
  • Update webhook URL in EaaS rule to ngrok URL
  • Send test email

Flow 5

Error Flows

Handle common error scenarios across the inbound email system

Personas: Various (Alex, Sarah, System)
Sub-flows: 4

5a: DNS Verification Failure

User adds MX records but verification fails

User adds MX records → Click Verify
DNS check fails
MX Not Found
Error: MX record not found
  • "MX record for example.com not detected"
  • Check DNS provider, verify record values
  • Wait for propagation
  • Link: Provider-specific guides
Wrong MX Value
Error: Value mismatch
  • "MX record found but value doesn't match"
  • Show expected vs actual values
  • "Copy correct value" button
Conflicting MX
Error: Multiple MX records
  • "EaaS must be the only MX"
  • Remove existing MX records
  • Or use a subdomain instead
