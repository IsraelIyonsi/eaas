Design Notes

Responsive Considerations

  • Sidebar: Collapses to a hamburger menu on screens below 768px. Icon-only mode available at 1024px breakpoint.
  • Email List Table: Transforms to a card-based list on mobile. Each card shows From, Subject, Status, and Time. Columns like Webhook Status and Attachments move inside the card as secondary details.
  • Analytics Dashboard: Stat cards stack 2x2 on tablet, 1-column on mobile. Charts resize responsively. Two-column layout (Status Breakdown + Top Senders) stacks vertically on smaller screens.
  • Setup Wizard: Stepper becomes vertical on mobile with current step highlighted. Action buttons stack vertically with primary action on top.
  • Forms: All inputs become full-width on mobile. Radio groups stack vertically below 480px.

Interaction Notes

  • Real-time Updates: Email list polls every 30 seconds for new emails. New arrivals animate in at the top with a subtle highlight. Analytics refresh every 60 seconds.
  • Optimistic UI: Rule toggle (active/inactive) applies immediately in the UI. If the API call fails, the toggle reverts with an error toast notification.
  • Bulk Actions: Selecting multiple emails via checkboxes reveals a floating action bar at the bottom of the table with options: Delete, Retry Webhook, Export.
  • Table Sorting: All columns in the email list are sortable. Default sort is "Received" descending (newest first). Sort state persists in URL query params.
  • Webhook Test: In the setup wizard and rule editor, "Test Webhook" sends a sample payload and shows the HTTP response code and latency inline below the button.

Accessibility Notes

  • Keyboard Navigation: All interactive elements are reachable via Tab key. Table rows are focusable with Enter/Space to open detail view. Escape closes modals and returns focus to the trigger element.
  • Screen Reader Labels: Status dots include aria-label (e.g., "Status: Processed"). Toggles announce their state ("Active: On/Off"). Charts include sr-only summary text describing the data.
  • Focus Management: On page transitions (e.g., list to detail), focus moves to the page heading. Error banners receive focus when they appear. Modal traps focus within the dialog.
  • Color Independence: Status is conveyed through both color and text labels. Badges include text in addition to visual styling. Chart data is also available in tabular form.
  • Contrast: All text meets WCAG AA minimum contrast ratios (4.5:1 for normal text, 3:1 for large text).

Screen-by-Screen UX Decisions

  • Screen 1 (Email List): Filters are additive and URL-persisted for shareability. Relative timestamps ("2m ago") improve scannability over absolute dates. Webhook status column provides at-a-glance delivery health without navigating to detail.
  • Screen 2 (Email Detail): Authentication verdicts (SPF, DKIM, DMARC) are shown prominently because they are critical for debugging delivery issues. Tabs for body content prevent information overload while keeping all data accessible.
  • Screen 3 (Thread View): Visual direction indicators (arrows, color tints) make it immediately clear which emails were sent vs received. Timeline connector creates a clear narrative flow.
  • Screen 4 (Rules): Priority numbers are explicit rather than hidden to help users understand evaluation order. Test webhook button prevents saving broken configurations.
  • Screen 5 (Setup Wizard): "Save and Continue Later" at every step respects that DNS propagation can take hours. Provider-specific guides reduce friction at the most error-prone step (MX records).
  • Screen 6 (Analytics): KPI cards at the top give immediate health signal. Percentage changes provide trend context. Top Senders helps identify patterns or potential abuse.
  • Screen 7 (Notifications): Inline threshold editing (not a separate form) reduces clicks. Per-domain configuration prevents alert noise from low-priority domains.
