Create Inbound Rule

POST /api/v1/inbound/rules

Creates a new inbound routing rule for a verified domain. Rules determine how incoming emails are processed and where they are forwarded.

Parameters

Name Type Description
name Required string A human-readable name for this rule (e.g., "Support Inbox")
domainId Required string The ID of the verified domain this rule applies to
matchPattern Required string Email pattern to match (e.g., "support@", "*@", "billing@")
action enum Action to take: webhook, store, or forward. Default: webhook
webhookUrl string URL to receive webhook notifications (required when action is webhook)
priority integer Rule priority (0 = highest). Default: 0

Response

201 Created

GET /api/v1/inbound/rules

Webhook Integration

EaaS delivers inbound email events to your application via HTTP webhooks. When an email is received and processed, EaaS sends a POST request to your configured webhook URL with the full email payload.

EaaS Platform

Payload Schema

Every webhook request includes a JSON body with the following structure:

Webhook Payload
{ "event": "inbound.email.received", "timestamp": "2026-03-28T14:22:00Z", "data": { "id": "em_8x2k...", "from": { "email": "john@example.com", "name": "John Doe" }, "to": ["support@myapp.io"], "subject": "Need help with billing", "text": "Hi, I need help with...", "html": "<p>Hi, I need help with...</p>", "attachments": [ { "filename": "invoice.pdf", "contentType": "application/pdf", "size": 24576, "url": "https://api.eaas.dev/v1/..." } ], "headers": { /* raw headers */ }, "spamScore": 1.2, "ruleId": "r7b2c..." } }

Field Reference

Field Path Type Description
event string Event type. Always inbound.email.received
data.id string Unique email identifier
data.from.email string Sender email address
data.to string[] Array of recipient addresses
data.subject string Email subject line
data.text string Plain text body
data.html string HTML body (if available)
data.attachments object[] Array of attachment objects with filename, contentType, size, url
data.spamScore number Spam confidence score (0-10, lower is better)

Signature Verification

Every webhook includes an X-EaaS-Signature header. Verify this signature to ensure the request is authentic.

Node.js Python C# Go
const crypto = require('crypto'); function verifySignature(payload, signature, secret) { const expected = crypto .createHmac('sha256', secret) .update(payload) .digest('hex'); return crypto.timingSafeEqual( Buffer.from(signature), Buffer.from(`sha256=${expected}`) ); }

Best Practices

  • 1 Return 200 quickly. Acknowledge the webhook within 5 seconds. Process the email asynchronously in a background job.
  • 2 Verify signatures. Always validate the X-EaaS-Signature header to prevent spoofed requests.
  • 3 Handle duplicates. Use the data.id field for idempotency. The same event may be delivered more than once.
  • 4 Use HTTPS. Your webhook endpoint must use TLS. HTTP endpoints will be rejected.
  • 5 Log everything. Store raw webhook payloads for debugging and auditing purposes.
  • 6 Monitor failures. Set up alerts for webhook delivery failures using the Notification Preferences page.

Inbound Email Architecture

Understand how email flows from sender to your application through the EaaS platform.

Prerequisites

  • ✓ An EaaS account with an active API key
  • ✓ A domain you own with access to DNS settings
  • ✓ A publicly accessible HTTPS endpoint for webhooks
  • ✓ Basic understanding of MX records and DNS configuration
⏱
Time to first email: ~30 minutes
Most of the time is spent waiting for DNS propagation (5-15 minutes).

1 Add Your Domain

Register your domain with EaaS via the dashboard or API. This generates the MX records you need to configure.

API
curl -X POST https://api.eaas.dev/v1/domains \ -H "Authorization: Bearer eaas_sk_..." \ -d '{ "domain": "example.com" }'

2 Configure DNS Records

Add the MX records provided by EaaS to your domain's DNS settings. The exact steps depend on your DNS provider.

Required DNS Records
# MX Record Type: MX Name: @ (or subdomain) Priority: 10 Value: inbound-smtp.us-east-1.amazonaws.com # SPF Record (TXT) Type: TXT Name: @ Value: v=spf1 include:amazonses.com ~all

3 Set Up Webhooks

Configure a webhook URL where EaaS will deliver parsed email payloads. You can do this via the Setup Wizard or the API.

4 Create Routing Rules

Define rules that determine how incoming emails are processed. Rules match against recipient addresses and can trigger webhooks, store emails, or forward them.

5 Send a Test Email

Send an email to your configured domain and verify it arrives at your webhook endpoint. Use the dashboard's test feature for instant verification.

DNS Provider Guides

Cloudflare ▼
  1. Log in to Cloudflare and select your domain
  2. Go to DNS > Records > Add Record
  3. Select Type: MX, Name: @, Mail server: inbound-smtp.us-east-1.amazonaws.com, Priority: 10
  4. Add a TXT record for SPF: v=spf1 include:amazonses.com ~all
  5. Save and wait 2-5 minutes for propagation
Amazon Route 53 ▼
  1. Open the Route 53 console and select your hosted zone
  2. Click Create Record > Simple routing
  3. Record type: MX, Value: 10 inbound-smtp.us-east-1.amazonaws.com
  4. Add a TXT record for SPF
  5. Changes typically propagate within 60 seconds
GoDaddy ▼
  1. Log in to GoDaddy and go to My Products > DNS
  2. Delete any existing MX records
  3. Add MX record: Host: @, Points to: inbound-smtp.us-east-1.amazonaws.com, Priority: 10
  4. Add TXT record for SPF verification
  5. Allow up to 48 hours for full propagation
Namecheap ▼
  1. Log in to Namecheap and go to Domain List > Manage
  2. Navigate to Advanced DNS
  3. Add a Mail Exchange (MX) record with the provided values
  4. Add a TXT record for SPF
  5. DNS changes may take 30 minutes to propagate

Troubleshooting FAQ

MX records not verifying? ▼

DNS changes can take up to 48 hours to propagate globally, though most providers update within 5-15 minutes. Use dig MX yourdomain.com to check propagation status. Ensure you removed any conflicting MX records from your DNS provider.

Webhooks not being delivered? ▼

Verify your endpoint returns a 2xx status code within 30 seconds. Check that your URL uses HTTPS and is publicly accessible. Review the webhook delivery logs in the EaaS dashboard for detailed error information.

Emails are being marked as spam? ▼

Ensure your SPF, DKIM, and DMARC records are correctly configured. EaaS provides a spam score for each email - you can use routing rules to filter emails above a certain spam threshold. Check the Analytics dashboard for spam score distribution.

How do I handle large attachments? ▼

EaaS supports attachments up to 25MB. Attachments are stored in S3 and the webhook payload includes a signed URL for downloading. URLs expire after 24 hours - use the API to generate fresh download links when needed.
